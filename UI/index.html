  • stuff
  • Check if there are tables used for layouts or other purpose other than tabular data. Screen reader inteprets everything with this html tag as a table, and reads the number of rows, columns, and data inside it.
  • Other than images, we need to describe better the links, in order for them to be easily spotted and better interpreted by screen readers.
  • Each area of the site needs roles: header, navigation(
    ), main content footer, for easier screen reading.
  • Screen readers have issues with content loaded dynamically.
  • Tab index all areas for easier tab navigation. (not just inputs / buttons)
  • make hyperlinks visible, or keep underline / create an alternative view for them. This is especially for color blind.
  • Definition list (dd dt dl) used for list elements(need to check the dropdowns as I recall I used this tags) The definition list is used by the screen readers as a caption regarding list elements.
  • Wrap checkboxes, radiobuttons in fieldsets, so the label and input be correctly intrepreted by screen readers
  • Use legend attribute in form elements
  • Use captions in tables
  • For more descriptive alternative text, use ‘longdesc’ attribute
  • Check scripting of elements (such as span as a link, div a a link)
  • No inputs should do a specific action just by focusing on it, without at least warning the user before the event happens, and no dropdown should do specific actions just by selecting the item.
  • On complicated formats (eg: mm/dd/yyyy input) or functions, we should give a clear example or hints.
  • basically keep the html syntax as close as possible to it’s standard. This is the best thing we can do regarding accesibility.
  • ARIA ARIA ARIA. Read about it, and use it's syntax on custom created elements
